What changed
- Replaced separate mailto CTAs with one logic-based intake on OUT-MATCH.
- Added one mobile-first intake for:
  - buyer

  - buy + sell
  - future OUT Agent
- Added expert application questions for:
  - Tesla owners
  - former Tesla employees
  - hybrid owner + former Tesla employee backgrounds
- Added buyer/seller/expert track logic in script.js.
- Added copy-summary fallback plus draft-email generation.
- Did not change styles.css.

How it works
- User picks a track.
- JS shows only the relevant questions.
- Form builds a structured summary.
- User can copy the summary or open a draft email to hello@onlyusedtesla.com.
- Query-string links can pre-select the form:
  - out-match.html?track=buyer#intake
  - out-match.html?track=seller#intake
  - out-match.html?track=both#intake
  - out-match.html?track=expert#intake

Recommended next step after this pilot
- Replace the draft-email flow with a real endpoint (Formspree, Web3Forms, Airtable, Supabase, etc.) once you want submissions stored automatically.
